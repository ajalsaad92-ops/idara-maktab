# تحويل التطبيق إلى نظام داخلي (Offline / LAN)

## الواقع الذي يجب فهمه أولاً

التطبيق حاليًا مبني بالكامل على **Lovable Cloud (Supabase السحابي)**:
- المصادقة (تسجيل الدخول، الجلسات، JWT) → خدمة Supabase Auth السحابية
- قاعدة البيانات → Postgres على Supabase Cloud
- صلاحيات الصفوف (RLS) → سياسات داخل Supabase
- الواجهة → منشورة على Cloudflare Workers (يحتاج إنترنت)

**التحويل إلى offline يعني استبدال كل هذه الطبقات** — ليس مجرد تغيير عنوان قاعدة بيانات. هذا عمل بحجم **عدة أيام/أسابيع**، وليس تعديلًا في رسالة واحدة.

## الحل المقترح (الأنسب لحالتك)

نختار جهاز كمبيوتر واحد من الأجهزة العادية ليكون "السيرفر" (يعمل دائمًا أو أثناء الدوام)، والبقية يتصلون به عبر الشبكة الداخلية بعنوان IP مثل `http://192.168.1.50:3000`.

### المكونات الجديدة

```text
┌─────────────────────────────────────────────────┐
│   جهاز "السيرفر" (كمبيوتر عادي - Windows)       │
│                                                  │
│  ┌──────────────┐    ┌──────────────────────┐  │
│  │  PostgreSQL  │←──→│  Node.js Server      │  │
│  │   محلي       │    │  (Express + JWT)     │  │
│  └──────────────┘    └──────────┬───────────┘  │
│                                  │              │
│                          منفذ 3000              │
└──────────────────────────────────┼──────────────┘
                                   │ LAN
        ┌──────────────────┬───────┴──────┬───────────────┐
        ↓                  ↓              ↓               ↓
   جهاز موظف 1       جهاز موظف 2    جهاز المدير    جهاز الاستقبال
   (متصفح فقط)        (متصفح فقط)    (متصفح فقط)   (متصفح فقط)
```

## مراحل العمل

### المرحلة 1: تجهيز البيئة المحلية
- تثبيت PostgreSQL 16 على جهاز السيرفر (مثبت Windows عادي)
- إنشاء قاعدة بيانات `idara_maktab` وتطبيق كل المخططات (Schema) الموجودة من ملفات `supabase/migrations/`
- تهيئة بيانات أولية من ملفات `supabase/seed/`

### المرحلة 2: استبدال طبقة المصادقة (الأصعب)
- إزالة `@supabase/supabase-js` من المتصفح
- بناء خدمة مصادقة محلية: JWT + bcrypt + جدول `users` + `sessions`
- نقطة `/api/auth/login`, `/api/auth/logout`, `/api/auth/me`
- استبدال `useAuth()` ليعمل مع الـ JWT المحلي بدلًا من Supabase

### المرحلة 3: استبدال طبقة البيانات
- إنشاء عميل قاعدة بيانات مباشر (pg أو Drizzle ORM)
- استبدال كل `supabase.from(...).select()` بطلبات REST لخادم Node.js محلي
- نقل منطق RLS من Postgres إلى middleware في Node.js (فحص الصلاحيات لكل طلب)
- ~30+ ملف في `src/components/pages/` و`src/lib/` يحتاج تعديل

### المرحلة 4: تبديل بنية النشر
- إزالة اعتماد TanStack Start على Cloudflare Workers
- إعداد `node server.js` يخدم الـ SPA + REST API على نفس المنفذ
- تعطيل SSR (لأن البيئة محلية ولا حاجة له) → SPA فقط
- إعداد build ينتج مجلد `dist/` ثابت + ملف server.js

### المرحلة 5: حزمة التثبيت
- سكربت `install.bat` للويندوز يقوم بـ:
  1. تثبيت Node.js (مضمّن)
  2. تثبيت PostgreSQL (مضمّن أو من المستخدم)
  3. تشغيل ملفات الـ migrations
  4. إنشاء خدمة Windows تشغّل السيرفر تلقائيًا عند الإقلاع
- ملف `README-التثبيت.md` بالعربية يشرح:
  - كيف يجد IP الجهاز السيرفر
  - كيف يفتح المنفذ في Windows Firewall
  - كيف يدخل الموظفون من أجهزتهم (`http://<IP>:3000`)

### المرحلة 6: ميزات Offline داخل المتصفح
- إزالة كل أيقونات/منطق "متصل بالإنترنت"
- التأكد من عدم وجود أي طلبات خارجية (خرائط Google، CDN خطوط، إلخ)
- تحميل خطوط Cairo/Inter محليًا من `public/fonts/`

## ما الذي سيتغير من تجربة الاستخدام

| قبل (Cloud) | بعد (LAN) |
|---|---|
| `https://id-maktab.lovable.app` | `http://192.168.1.50:3000` |
| تسجيل دخول بإيميل من أي مكان | تسجيل دخول من داخل الشبكة فقط |
| نشر تلقائي من Lovable | تحديث يدوي بنسخ مجلد `dist/` |
| نسخ احتياطي تلقائي | يجب جدولة `pg_dump` يدويًا |
| لا يحتاج جهاز مخصص | جهاز السيرفر يجب أن يبقى مشغّلًا |

## ما أحتاجه منك للبدء

1. **تأكيد**: هل تريد أن أبدأ فعلًا بهذا التحويل الكبير؟ المشروع الحالي على Cloud سيتوقف عن العمل عند الانتهاء (أو نحتفظ بنسختين).
2. **نظام جهاز السيرفر**: Windows 10/11 أم Windows Server؟
3. **عدد المستخدمين المتوقع**: 5؟ 20؟ 50؟ (يؤثر على إعدادات Postgres)
4. **هل يوجد إنترنت في المكتب على الإطلاق؟** (مفيد لتحديث التطبيق لاحقًا، أو نحتاج وضع تحديث عبر USB)

## بديل أبسط (لو أردت)

لو الهدف الأساسي هو **عدم الاعتماد على إنترنت** لكن لديك واي‌فاي عادي في المكتب: نبقى على Lovable Cloud ونضيف **وضع PWA + Cache** بحيث يعمل التطبيق دون إنترنت لفترات قصيرة ويزامن لاحقًا. هذا يستغرق ساعة واحدة بدلًا من أسابيع.
