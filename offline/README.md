## نظرة عامة

```text
┌──────────────────────────────────────────┐
│   جهاز "السيرفر" (Windows عادي)          │
│                                           │
│   PostgreSQL  ←──→  Node.js (المنفذ 3000) │
└──────────────────┬───────────────────────┘
                   │ شبكة WiFi/LAN المكتب
       ┌───────────┼───────────┬─────────────┐
       ↓           ↓           ↓             ↓
   موظف 1      موظف 2      المدير      الاستقبال
   (متصفح)     (متصفح)     (متصفح)      (متصفح)
```

- **جهاز السيرفر**: كمبيوتر واحد فقط يكون مشغّلًا أثناء الدوام، يحتوي قاعدة البيانات والخادم
- **بقية الأجهزة**: تحتاج فقط متصفح (Chrome / Edge / Firefox) لا تثبيت برامج

## المتطلبات

### على جهاز السيرفر فقط:
- Windows 10 أو Windows 11 (أو Windows Server)
- ذاكرة 4 GB كحد أدنى (8 GB موصى به لـ 20+ موظف)
- مساحة فارغة 5 GB

### على أجهزة الموظفين:
- متصفح حديث (Chrome 100+ أو Edge 100+)
- متصلين بنفس شبكة WiFi/LAN

## التثبيت على جهاز السيرفر

### الخطوة 1 — تثبيت Node.js
1. حمّل **Node.js LTS** من https://nodejs.org/ (نسخة 20 أو أحدث)
2. شغّل المثبّت واترك كل الخيارات الافتراضية
3. للتأكد افتح PowerShell واكتب: `node --version`

### الخطوة 2 — تثبيت PostgreSQL
1. حمّل **PostgreSQL 16** من https://www.postgresql.org/download/windows/
2. شغّل المثبّت:
   - **كلمة سر المسؤول (postgres)**: اختر كلمة سر قوية واحفظها (مثل `MyOffice@2026`)
   - **المنفذ**: اتركه `5432`
   - **اللغة (Locale)**: `Arabic, Iraq` أو `Default locale`
3. عند انتهاء التثبيت لا تحتاج فتح Stack Builder — اضغط Cancel

### الخطوة 3 — نسخ ملفات التطبيق
1. انسخ مجلد `offline/` كاملاً إلى `C:\IdaraMaktab\`
2. افتح PowerShell **كمسؤول** داخل المجلد:
   ```powershell
   cd C:\IdaraMaktab
   ```

### الخطوة 4 — التهيئة التلقائية
شغّل ملف التثبيت بنقرة مزدوجة:
```
install.bat
```
سيقوم تلقائيًا بـ:
- تثبيت مكتبات Node.js
- إنشاء قاعدة بيانات `idara_maktab`
- تطبيق المخطط (Schema) والبيانات الأولية
- إنشاء حساب مدير افتراضي

عندما يُطلب منك كلمة سر postgres، أدخل التي اخترتها في الخطوة 2.

### الخطوة 5 — فتح المنفذ في جدار الحماية
شغّل PowerShell **كمسؤول** والصق:
```powershell
New-NetFirewallRule -DisplayName "IdaraMaktab" -Direction Inbound -Protocol TCP -LocalPort 3000 -Action Allow
```

## تشغيل التطبيق

### تشغيل يدوي
نقرة مزدوجة على `start.bat` ← سيفتح نافذة سوداء تعرض:
```
✅ السيرفر يعمل على http://0.0.0.0:3000
🌐 عنوان الشبكة: http://192.168.1.50:3000
```
**احفظ هذا العنوان** — هو الذي ستعطيه للموظفين.

### تشغيل تلقائي عند إقلاع الويندوز (موصى به)
1. اضغط `Win + R`، اكتب `shell:startup`، اضغط Enter
2. انسخ ملف `start.bat` إلى المجلد المفتوح (Startup folder)

سيعمل السيرفر تلقائيًا في كل مرة يُشغَّل الجهاز.

### معرفة عنوان IP الخاص بجهاز السيرفر
افتح PowerShell واكتب:
```powershell
ipconfig | findstr IPv4
```
ستجد سطرًا مثل: `IPv4 Address. . . . . . . . . . . : 192.168.1.50`

## دخول الموظفين من أجهزتهم

1. من أي جهاز متصل بنفس الشبكة، افتح المتصفح
2. اكتب في شريط العنوان (مع المنفذ):
   ```
   http://192.168.1.50:3000
   ```
   (استبدل `192.168.1.50` بعنوان جهاز السيرفر الفعلي)
3. ستظهر صفحة تسجيل الدخول

### الحساب الافتراضي بعد التثبيت
- **البريد**: `admin@office.local`
- **كلمة السر**: `Admin1234`
- ⚠️ **غيّر كلمة السر فورًا** من صفحة الإعدادات

## النسخ الاحتياطي

### نسخة احتياطية يدوية
شغّل `backup.bat` ← ينشئ ملف في `C:\IdaraMaktab\backups\backup-YYYY-MM-DD.sql`

### جدولة يومية (موصى به)
1. افتح **Task Scheduler** (`taskschd.msc`)
2. Create Basic Task → اسمها "نسخة احتياطية إدارة المكتب"
3. Trigger: Daily في وقت بعد انتهاء الدوام (مثل 10:00 مساءً)
4. Action: Start a program → اختر `C:\IdaraMaktab\backup.bat`

### استعادة نسخة احتياطية
```powershell
cd C:\IdaraMaktab
.\restore.bat backups\backup-2026-06-10.sql
```

## استكشاف الأخطاء

| المشكلة | الحل |
|---|---|
| "تعذر الاتصال بالخادم" من جهاز موظف | تأكد أنه على نفس الشبكة، وأن `start.bat` يعمل على السيرفر |
| "Connection refused" | تحقق من جدار الحماية (الخطوة 5 من التثبيت) |
| نسيت كلمة سر المدير | شغّل `reset-admin-password.bat` على السيرفر |
| السيرفر بطيء جدًا | أعد تشغيل الجهاز، أو زِد ذاكرة Postgres في `postgresql.conf` |
| ظهور خطأ "ECONNREFUSED 127.0.0.1:5432" | خدمة PostgreSQL متوقفة — افتح Services واشتغّل `postgresql-x64-16` |
